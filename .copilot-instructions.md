# TouchVault - Copilot Integration Guide

## What is TouchVault?

TouchVault is a PowerShell module that retrieves credentials from a KeePass database protected by a YubiKey hardware key. It provides secure, cached access to API keys, passwords, and secrets without ever storing them in plaintext.

## How to Use in Scripts

When a script needs credentials (API keys, passwords, client secrets), use TouchVault instead of hardcoding or prompting:

```powershell
# Import the module
Import-Module TouchVault

# Retrieve credentials by KeePass entry name
$creds = Get-VaultEntry "Entry Name Here"
$creds.UserName      # Client ID, API key, or username
$creds.Password      # Client secret, API key, or password
$creds.URL           # Service URL (optional)
$creds.Notes         # Metadata (optional)

# Or get a single field
$token = Get-VaultSecret "Entry Name" -Field Password
```

## Important Rules

1. **Never hardcode secrets** - Always use `Get-VaultEntry` or `Get-VaultSecret`
2. **Non-secret config** goes in JSON files (URLs, org IDs, resource IDs)
3. **Secrets** go in KeePass entries (API keys, passwords, tokens)
4. **First call** may require a YubiKey touch (popup appears automatically)
5. **Subsequent calls** use the cache (no touch needed for 8 hours)

## Standard Pattern for Script Integration

```powershell
# At the top of any script needing credentials:
Import-Module TouchVault

# Load non-secret config
$config = Get-Content '~\.myapp\config.json' -Raw | ConvertFrom-Json

# Load secrets from KeePass
$creds = Get-VaultEntry "KeePass Entry Name"

# Combine into a config object
$appConfig = [PSCustomObject]@{
    baseUrl      = $config.baseUrl
    clientId     = $creds.UserName
    clientSecret = $creds.Password
}
```

## Available Functions

- `Get-VaultEntry "Name"` - Returns hashtable with all fields
- `Get-VaultSecret "Name" -Field Password` - Returns one field
- `Clear-VaultCache` - Clears cached credentials (forces re-auth)
- `Update-VaultEntry "Name"` - Force-refresh one entry
- `Format-Masked $value` - Mask a secret for display
- `Test-VaultPrerequisites` - Check system status

## DO NOT

- Store passwords, API keys, or tokens in `.json`, `.ps1`, or any config file
- Use `ConvertTo-SecureString` with hardcoded strings
- Log or print unmasked secrets (use `Format-Masked` instead)
- Bypass the cache with `ForceRefresh` unless the credential was just rotated
